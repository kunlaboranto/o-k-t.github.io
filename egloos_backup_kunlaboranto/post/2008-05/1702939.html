delete후 ager가 동작해 실제로 데이터가 삭제되었는지는 다음의 쿼리를 이용하면됩니다.

iSQL> SELECT AGING_REQUEST_OID_CNT, AGING_PROCESSED_OID_CNT FROM V$MEMGC;
AGING_REQUEST_OID_CNT AGING_PROCESSED_OID_CNT
-------------------------------------------------
150071               150071              
150071               150071              
2 rows selected.

ager에게 요청한 것과 수행한 값이 같을 경우에는 ager가 모든 데이터를 삭제했음을 의미합니다.

Ager에서 모든 레코드를 삭제한 후 다시 compact 명령을 수행할 경우

삭제된 빈 페이지를 반환해 테이블 크기가 줄어들게 됩니다.
