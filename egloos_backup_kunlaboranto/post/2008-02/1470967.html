[oracle] alter session의 이상한 정책..

oracle에 비해 altibase의 alter session 정책이 불편하다는 생각에 테스트를 진행함..

altibase version : 4.3.9.100

isql> autocommit off;
isql> insert into demo_plan values (6000,1,1);
1 row inserted.
iSQL> alter session set explain plan=only;
[ERR-41047 : The transaction is already active.]


이를 oracle에서 테스트 해보니.
commit 여부와 alter session은 전혀 관련이 없음.

-- 1건 넣고 커밋 안된 상태에서..
SQL> ALTER SESSION SET SQL_TRACE = TRUE;
Session altered.

SQL> set autocommit on;
SQL>  commit;
Commit complete.    --> 신기한 것은 autocommit ON 상태에서 commit도, 이전 off에서 insert 한게 진짜로 commit 됨
SQL> set autocommit off;
SQL> insert into ok1 values (1);
1 row created.
SQL> set autocommit on;
SQL> rollback;         --> 역시 rollback도 잘 동작함.
Rollback complete.
