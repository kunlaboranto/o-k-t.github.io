명칭

footprints - 컴파일러의 footprint 레코드 정보의 요약

구문

/usr/ccs/bin/footprints[-V][-v] [-w]filename...

설명

footprints커맨드는, 지정한 object file을 스캔해, 그 파일에 격납 되어 있는 컴파일러footprint 레코드를 분석해, 요약 레포트를 생성합니다.이 레포트는, HP 서포트 및 연구실 기술자가 문제의 진단에 이용하는 것을 목적으로서 있습니다.

레포트에는, 이하의 정보가 포함됩니다.

  • 검출되었던 compilation unit의 총수

  • 최초와 최신의 컴파일러 time stamp와 링커 time stamp(존재할 경우)

  • text segment와 data segment의 총 사이즈

  • 사용되었던 컴파일러의 버전과, 각 버전으로 컴파일한 compilation unit 수의 리스트

  • 사용되었던 옵션과, 각 옵션 설정으로 컴파일한 compilation unit 수의 리스트.이 리스트는 2개의 그룹에 나뉘어, 가장 중요도가 높은 옵션이 최초의 그룹에 분류당해, 나머지의 옵션이 2번째의 그룹에 분류당해 그 후에 알파벳순으로 리스트 됩니다.

이하의 file type이 인식됩니다.

  • 실행 가능 파일과 공유 도서관

    footprints커맨드는, 모든 의존 도서관, 부정 시스템 공유 도서관을 검색해, 각 공유 도서관에 대해서 개별의 레포트를 출력합니다.LD_LIBRARY_PATH또는SHLIB_PATH환경 변수가 설정되고 있을 경우는, 환경 변수내에 리스트 되어 있는 디렉토리에서 의존 도서관을 검색합니다(LD_LIBRARY_PATH가 설정되고 있을 경우 ,SHLIB_PATH하 무시 당합니다).또, object file에 메워 넣어져 있는 검색 패스도 사용합니다.공유 도서관이 발견되지 않을 경우는, 진단 메세지를 출력하고 실행을 계속합니다.

  • 아카이브 도서관

    footprints커맨드는, 아카이브의 각 object file 멤버를 스캔합니다.

  • 재배치 가능 object file

    footprints커맨드는, 지정한 object file을 스캔합니다.

footprints커맨드는, SOM 및 ELF object file 형식, 32비트 및 64 bit code, PA-RISC 및 Integrity 시스템을 서포트 합니다.또, HP C 및 C++ 컴파일러가 생성한 컴파일러footprint을 서포트 많아진다.다른 컴파일러가 생성한 compilation unit은, 요약에 반영되지 않을 때가 있습니다.

옵션

footprints커맨드는, 이하의 옵션을 인식합니다.

-V

버전번호를 출력하고 종료됩니다.

-v

용장 모드: 검출한 각 compilation unit에 대해서, 상세한 정보를 출력합니다.

-w

상호교환성의 문제가 일어날 가능성에 대해서, 경고를 출력합니다.현재(에)는, 버전 A.05.50보다 앞의 C/ C++ 컴파일러에서 Integrity시스팀용으로 구축한 compilation unit 전부에 대해서 경고가 출력됩니다.경고는stderr니 출력되어, 유쿠토우에게는 아스타리스크가 3개 부가 됩니다.

사례

다음의 사례는, 일반적인 요약 레포트의 선두부분입니다.

version행은, ANSIC컴파일러의 버전A. 06. 11로 합계 51개의 compilation unit이 컴파일된 것을 나타냅니다.architecture행은, Integrity시스템 (Itanium(R) Product Family) 용으로 어플리캐이션이 32 bit data 모델로 컴파일된 것을 나타냅니다.

-O행은, 1개의 compilation unit이 default의 최적화 레벨(레벨 01)로 컴파일되어, 그 외의 50개는 레벨 02로 컴파일된 것을 나타냅니다.PBO행에 의하면, 51개의 유니트는 전부, 프로필 베이스의 최적화를 사용하지 않고 컴파일되어 있습니다.

-g행은, 기호디버깅 옵션을 사용하고 컴파일된 compilation unit이 1개도 없는 것을 나타냅니다.기호디버깅 옵션을 사용하고 컴파일된 compilation unit이 있을 경우에는 ,debugopt행에는, 기호디버깅 및 높은 최적화 레벨을 다양하게 맞추고 컴파일한 compilation unit의 수가 표시됩니다.

-mt행과 같이 on/ off 옵션 설정이 각각 안에 리스트 되어 있을 경우, 디펄트 설정을 compiler option에 따라서(의해서) 옮겨놓을 수 없는 것을 나타냅니다.

footprint 레코드내의 옵션에는, 커맨드행의 컴파일러 스위치와 직접 대응하고 있지 않는 것이나, 지철기의 다른 것이 많이 있습니다.이들은, HP 서포트 및 연구실 기술자가 사용할 수 있도록, 레포트에 포함되어 있습니다.서포트 되어 있는 커맨드행 스위치에 대해서는 ,cc(1)오 참조해 주세요.
