Unified File Cache

Unified File Cache(UFC)는 페이지 캐시와 버퍼 캐시를 통합하여파일 액세스에 대해 일관성을 제공합니다. 현재 파일 시스템은 버퍼캐시를 사용하여 파일 데이터를 캐시하고 mmap() 인터페이스는 페키지 캐시를 사용하여 파일 데이터를 캐시합니다.응용 프로그램에서 read(2)/write(2) 시스템 호출과 mmap(2)를동시에 사용하여 파일에 액세스하면 데이터가 두 캐시에 모두 있으므로HP-UX에서 일관성을 보장하지 않습니다. Unified File Cache를 사용하면일관성을 유지할 수 있습니다. 파일 데이터를 액세스하는 아키텍처를통합하면 파일 시스템에서 read() 및 write() 시스템 호출과 mmap()에 대해 동일한 커널 인터페이스를 사용합니다. 이러한 공통인터페이스는 Solaris에서 타사 파일 시스템의 이식을 편리하게 합니다.

영향

HP-UX 11i v3의 Unified File Cache는 고객에게 다음과 같은 이점을제공하고 가치를 높입니다.

  • HP-UX 11i v3의기본 요구 사항인 VxFS 4.1 및 ONC+2.3의 주요 enabler 역할을 합니다.

  • 페이지 및 버퍼 캐시 일관성에종속된 Solaris, Tru64 및 Linux 응용 프로그램과의 소스 호환성을 향상시킵니다.

  • 또한 페이지 및 버퍼 캐시일관성에 종속된 응용 프로그램의 잠재적 성능이 향상됩니다.

호환성

알려진 호환성 문제는 없습니다.

성능

페이지 및 버퍼 캐시 일관성에 종속된 응용 프로그램의 잠재적성능이 향상됩니다. 이러한 성능 향상은 응용 프로그램에 따라 달라집니다.
