[팁] sqlplus의 &1을 isql에서는

1. 현상

현재 altibase의 isql은

sqlplus의 편리한 기능인 &1, &2, &<변수명>, &&<변수명> 등을 지원하고

있지 못하므로 불편함.

 

4. 코드 수정전 대응방법

[ 예 ]

#!/bin/sh

 

SQL1="

select

    (

    (select count(\*) from A_TABLE_NAME)

    \* ceil((offset+size)/8)\*8

    / 1024 / 1024

    )

from adm_tb_size

where table_name = 'A_TABLE_NAME'

order by offset desc

limit 1

;"

 

#echo $1

echo "["$SQL1"]"

 

echo $SQL1 | sed -e "s/A_TABLE_NAME/$1/g" |sed -e "s/\\\\//g" > .tmp.sql

 

is -f .tmp.sql

 

+++++++++++++++++++++=======

이 view도 추가되었으면 좋겠습니다.

 

sizing할때 제가 자주 사용하는 쿼리입니다. fixed일때만 유용하지만..

 

create view adm_tb_size as

  select table_name, column_name, offset, size

  from system_.sys_tables_ a, system_.sys_columns_ b

  where a.table_id = b.table_id;

 

이 뷰가 있으면 사이즈 구할 때 다음 쿼리로 가능..

 

select * from adm_tb_size where table_name = 'T1';
