TIMEOUT=3;CONNECTION_TIMEOUT=10

SESC에서 reconnect 하는 예제 입니다.


1] 설정예

예)

    strcpy(conn_opt, "DSN=192.168.3.230;PORT_NO=20300; CONNTYPE=1;TIMEOUT=3;CONNECTION_TIMEOUT=10");

    EXEC SQL CONNECT :usr IDENTIFIED BY :pwd USING :conn_opt;

   -->  TIMEOUT : 서버 연결 시도 시 기다리는 시간. 기본값은 3초입니다. 

        CONNECTION_TIMEOUT : 네트워크 불안정 시 select() 또는 poll() 에서 발생할 수 있는 blocking을 방지하기 위한 속성입니다.

2] 주의 사항

 CONNECTION_TIMEOUT 은 네트웍 오류시 빠른 failover를 위한 설정으로 10초로 설정이 된 경우

   어떤 쿼리가 lock wait으로 10초이상 대기되거나 원래 10초 이상 수행되는 쿼리라면 ( 정확하게는 10초이내에 server 단에서 응답을 받지 못하는 쿼리 )
   CONNECTION이 끊어지게 됩니다.

   단 SELECT의 경우 FETCH 중 전체 시간을 의미하는 것은 아니며 보통 3000건 단위로 서버와 통신을 하게 되는데

   이때 서버와 통신하는 간격사이의 시간을 의미합니다.

3] 테스트 결과

   RECONNECT을 처리하는 로직에서

   4.3.9.6 버전으로 테스트시 “(a) EXEC SQL DISCONNECT;” 나 “(b) EXEC SQL FREE;” 를 둘 중 1개 혹은 둘 모두 설정한 경우에

   모두 정상적으로 reconnection이 되었습니다.

   샘플 소스상에서는 (a) -> (b) 으로 2번 호출 하도록 권장합니다.
