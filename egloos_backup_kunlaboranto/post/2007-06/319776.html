오늘 2006년 11월 SQL*Box를 기능면에서 upgrade를 진행하였습니다. 몇몇분의 요청사항을 이유있다고 판단하여
미뤄오던 작업을 진행했습니다. 관련 내용도 help file(sqlbox.hlp)에 추가되었구요.
간단히 말씀드리면 "SQL*Box" 창에서 SQL을 작성한 후 마우스로 drag하여 마킹한 부분만 따로 수행이 가능하도록
설정을 했습니다. 마우스로 마킹하시고 "F2" 혹은 "CTRL+ENTER"인 경우만 작동을 합니다. 마우스 오른쪽 버튼이나
파일 메뉴의 "Execution"은 이 기능이 없음을 유의하시기 바랍니다. 또한 "desc table_name"을 매번 작성하여 조회
하는 것도 불편하다는 생각이 들어서 단축키로 처리했습니다. 이제 아무 위치에서나 커서를 원하는 테이블이름에
놓은 후 단축키로  "F3"을 누르시면 해당 테이블의 description 정보를 보여주도록 하였습니다.
계속되는 여러분들의 성원에 감사드립니다.

CF. 잠깐.. v2.6.2 patch가 막 진행되었습니다.(11월 15일)

 index 정보추출시 다른 사용자의 object(index)에 대하여 read권한 만 있는 경우 object information을 access할 수 없었습니다.
 이 경우 error로 처리하여 sqlbox에서 다른 정보가 더이상 나타나지 않았는데 이 버젼에서는 권한이 없어서 안나오는 부분은 그대로 error표시하고
 다음단계의 정보는 계속 보여주도록 설정하였습니다.
 상단의 데이터베이스 version 정보는 컴포넌트와 레지스트리로 구성이 되어있는바 현재는 레지스트리 권한이 없으면 컴포넌트 정보도 나타나지 않고있습니다.
 이번 버젼에서는 레지스트리 권한이 없는 사용자의 경우 컴포넌트 정보만이라도 볼 수 있도록 하였습니다.
 Tip. version 정보 전체를 보고 싶은 경우 컨트롤키를 누른 상태에서 마우스 오른쪽 버튼으로 version이 표시되는 창을 클릭하시면 해당 정보가 clipboard로 카피됩니다.

 dwmenu 변경 : data 조회 후 조회된 창에서 마우스 오른쪽 버튼으로 조회된 data를 copy하는 방식에 all 컬럼 copy를 추가하였습니다.
                       모든 rows를 copy하되 현재 위치에 해당되는 컬럼 data만 copy하는 기능이 추가되었습니다. (아직 help에는 설명이 추가되지 않았습니다)
                       또한 이 dwmenu에서 copy하는 data들은 마지막 문자가 tab이나 newline character가 있어 불편함이 있다는 지적이 있었습니다.
                       이에 copy된 data중 마지막 tab 혹은 newline character를 제거하여 불필요한 editing 작업을 없애도록 함

Upgrade는 그냥 download하시고 overwrite하시면 됩니다.

아시다시피 현재까지는 freeware이며 powerbuilder로 만들어져 있습니다.
(부가적인 보안 기능을 원하는 경우에는 각 회사에 맞게 프로그램을 수정하는 유료화 정책이 있습니다만 현재 SQL*Box가 제공하는 자체 기능은 모두 무료입니다)

사용 조건은 SQL*Net(또는 oracle net)이 있어야 합니다. 하지만 현재 이 tool을 만든 powerbuilder version8의 한계로 인하여
oracle10g driver가 없기 때문에 일부 oracle10g 고유의 SQL과 같은 것은 지원되지 않는 경우도 있습니다. 그러나 대부분 작업을
하는데 있어서 별다른 문제는 없을 것입니다.

구성은 총 2가지입니다.

1. SQL*Box tool

2. SQL*Box내 database 분석부분을 위해 사용되는 package set과 분석하는 방법을 담은 문서
   (이 부분은 DBA가 아니시면 필요없습니다)

위 그림에서 보듯 registry를 열어서 (윈도우 시작 버튼 --> 실행 --> regedit) "HKEY_LOCAL_MACHINE" --> SOFTWARE --> ORACLE"에서
HOME0 폴더의 오른쪽 KEY 값 중 ORACLE_HOME의 설정이 제대로 되어있는지 확인하시기 바랍니다.

만일, 위 값들이 제대로 되어있지 못하면 oracle client를 제대로 찾지 못할 수 있습니다. 만일, 여러 차례에 걸쳐 oracle client 혹은 oracle을 install
하셨던 분들은 위에서 왼쪽 밑에 HOME0외에도 HOME1, HOME2 이런 식으로 계속 폴더가 있을 수 있습니다. 이 경우엔 HOME?중 가장 큰 값
즉, HOME0 ~ HOME9 까지 있다면 HOME9가 되겠지요. 이 폴더를 선택한 후 오른쪽 KEY 값으로 ORACLE_HOME을 제대로 설정하면 됩니다.
